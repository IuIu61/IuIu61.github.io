SAÉ 1.05 Compétence RT3 · Coeff. 35

Traiter des données

Développement en binôme d'un script Python pour collecter automatiquement des données via une API REST OpenStreetMap, les traiter statistiquement et produire un rapport exporté au format Markdown/HTML.

Python API REST JSON / XML OpenStreetMap Markdown Bibliothèque requests Statistiques
01

Contexte & Objectifs

Problématique professionnelle : Le professionnel R&T est régulièrement amené à traiter des données issues du système d'information de l'entreprise (état des équipements, utilisateurs, logs réseau). Ces données brutes doivent être traitées, nettoyées et présentées de façon synthétique, idéalement de manière automatisée via des scripts.

Mission confiée : En binôme, créer un programme Python capable de : (1) récolter des données via l'API REST OpenStreetMap, (2) mettre en forme les données selon les informations désirées, (3) effectuer des traitements statistiques, (4) exporter les résultats au format Markdown (et optionnellement en HTML via la bibliothèque Python markdown).

Organisation : Projet en binôme sur 13h à 20h en autonomie, avec deux points d'étape évalués (mi-projet et fin de projet). Note finale = moyenne entre note de projet et note de suivi.

02

Compétences techniques mobilisées

  • API REST et protocole HTTP : Compréhension du modèle requête/réponse REST, codes de statut HTTP (200, 404, 500…), paramètres d'URL, clés d'authentification.
  • Formats de données JSON et XML : Parsing et manipulation de données structurées avec Python (bibliothèques json, requests).
  • Programmation Python : Listes, dictionnaires, fonctions, boucles, gestion de fichiers, traitement statistique des données collectées.
  • Format Markdown : Syntaxe de balisage légère (titres, gras, listes, tableaux, images), compilation en HTML via la bibliothèque Python markdown.
  • Ressources mobilisées : R106, R107 (programmation), R108 (systèmes), R109 (technologies web), R115 (gestion de projet).
03

Ma contribution & tâches réalisées

À compléter : décrivez le jeu de données choisi sur OpenStreetMap (quelles données ? quelle zone géographique ?), la structure de votre script Python, les traitements statistiques effectués, le format du rapport Markdown produit et la répartition des tâches dans le binôme. Mentionnez les deux points d'étape et les retours obtenus.

04

Traces & livrables

  • Script Python commenté et fonctionnel (collecte + traitement + export)
  • Fichier de rapport généré au format Markdown (.md)
  • Fichier HTML produit par compilation du Markdown (optionnel)
  • Évaluation des deux points d'étape intermédiaires
05

Analyse réflexive

Difficultés rencontrées : À compléter — par exemple : compréhension de la structure JSON renvoyée par l'API OSM, gestion des erreurs HTTP, structuration du code Python en fonctions réutilisables, mise en forme du rapport Markdown.

Solutions apportées : À compléter — lecture de la documentation de l'API OSM, tests progressifs avec un petit jeu de données avant de passer à l'ensemble, débogage des requêtes via le navigateur.

Ce que j'ai appris : À compléter — compréhension du modèle REST et de son usage dans les SI, automatisation du traitement de données en Python, production de rapports lisibles par des humains et des machines, organisation du travail en binôme sur un projet de programmation.

Positionnement personnel : À compléter — évaluez votre niveau de maîtrise de la compétence RT3 après cette SAÉ, notamment sur l'axe programmation et traitement de données.
