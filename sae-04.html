SAÉ 1.04 Compétence RT3 · Coeff. 8

Se présenter sur Internet

Concevoir et réaliser un site web multipage en HTML/CSS dans le cadre de la ressource R1.15 — Gestion de projet. Site responsive, conforme aux standards W3C et WCAG 2.0 AA, avec animations CSS et hébergement GitHub Pages.

HTML5 / CSS3 Flexbox / CSS Grid Responsive Design WCAG 2.0 AA Animations CSS GitHub Pages Gestion de projet
01

Contexte & Objectifs

Projet : Dans le cadre de la SAÉ 1.04 — «Se présenter sur Internet» — et de la ressource R1.15 «Gestion de projet : Maîtriser les bases», réaliser un site web multipage au thème libre (sujet personnel ou hobby). Ce projet constitue à la fois un livrable technique et un exercice de gestion de projet.

Cahier des charges : Le site devra comporter au minimum trois pages (accueil visuelle, page dédiée au sujet, page à propos), utiliser une charte graphique cohérente, passer la validation W3C (HTML et CSS), être adaptatif (tablette, mobile, ordinateur), intégrer des animations CSS et être déposé sur un repository GitHub avec hébergement via GitHub Pages.

Notation : Design et UX/UI (4 pts), contenus et rédactionnel (4 pts), respect du cahier des charges (6 pts), livrables R1.15 — WBS, rétroplanning, tableau de suivi (6 pts).

02

Exigences techniques ciblées

  • Structure HTML sémantique : Utilisation des balises HTML5 appropriées, arborescence de pages cohérente avec navigation fonctionnelle.
  • CSS avancé : Flexbox et/ou CSS Grid pour la mise en page, variables CSS, charte graphique homogène sur l'ensemble des pages.
  • Responsive Web Design : Affichage correct sur ordinateur, tablette et mobile (media queries).
  • Accessibilité WCAG 2.0 AA : Contrastes suffisants, attributs alt sur les images, navigation au clavier fonctionnelle.
  • Animations CSS : Au moins une animation CSS significative (transition, keyframes, transform).
  • Versioning et déploiement : Repository GitHub avec commits porteurs de sens et hébergement sur GitHub Pages.
03

Ma contribution & tâches réalisées

À compléter : décrivez le thème choisi pour votre site, l'organisation du projet (WBS, rétroplanning), les pages créées, les choix de design (palette, typographie, layout), les animations implémentées et les difficultés rencontrées lors de la validation W3C ou du déploiement GitHub Pages. Indiquez le lien vers votre site hébergé.

04

Traces & livrables

  • Site web multipage (≥ 3 pages) en HTML/CSS — lien GitHub Pages à renseigner
  • Repository GitHub avec historique de commits significatifs
  • README.md détaillé (présentation du projet, technologies, lien hébergement)
  • WBS ou MindMap de planification
  • Rétroplanning ou diagramme de Gantt
  • Tableau de suivi (Trello ou Kanboard)
05

Analyse réflexive

Difficultés rencontrées : À compléter — par exemple : maîtrise de CSS Grid sur des mises en page complexes, gestion de la compatibilité navigateur, validation W3C des formulaires, respect des contrastes WCAG.

Solutions apportées : À compléter — utilisation du validateur W3C en cours de développement, tests sur plusieurs navigateurs, ajustements progressifs des media queries.

Ce que j'ai appris : À compléter — maîtrise du cycle complet de création d'un site web, de la conception à la mise en ligne, gestion de projet en autonomie avec des livrables planifiés, importance de l'accessibilité et des standards web.

Positionnement personnel : À compléter — évaluez votre niveau de maîtrise de la compétence RT3 après cette SAÉ et les aspects à approfondir (JavaScript, frameworks CSS, etc.).
